제 1강 : About linux

 

리눅스란?

리눅스(Linux)는 리누스 토르발스가 개발한 컴퓨터 운영 체제(OS)입니다.

여러분들이 사용하고 있는 윈도우즈(Windows)와 비슷한 것이라고 보면 됩니다. 컴퓨터의 가장 기본적인 프로그램이 OS입니다.

리눅스를 배우는 이유는 다음과 같습니다.

  • 대부분의 서버는 리눅스로 돌아갑니다 (그리고 우리는 이런 서버에서 여러 작업을 할 것입니다)
  • 멋있어 보입니다 😘

농담입니다.

 

리눅스 기초 명령어

시작하기 전에, 혹시 GUI와 TUI가 뭔지 아시나요?

그래픽 사용자 인터페이스(graphical user interface, GUI)는 지금 여러분이 쓰고 있는 인터페이스입니다. 여러분이 게임을 하고 싶을 때 아이콘을 클릭하는 것도 GUI의 예시입니다. 쉽게 말해, 마우스로 클릭할 수 있다는 것입니다.

반면에, 텍스트 사용자 인터페이스(TUI, Text-based user interface)는 마우스를 쓰지 않는 대신 키보드만을 쓰는 인터페이스입니다. 영화나 드라마에서 해커들이 검은 화면에 글씨를 써서 컴퓨터를 쓰는 장면이 대표적인 텍스트 사용자 인터페이스의 예시입니다. 쉽게 말해, 마우스로 클릭할 수 없다는 것입니다.

아래와 같은 환경이 TUI입니다.

 

 

우리는 앞으로 TUI환경에서 학습을 진행할 것입니다.

 

그렇기에, 리눅스 TUI환경에서 필수적인 몇 가지 명령어들을 알아봅시다.

 

cd

먼저 파일과 폴더(디렉터리)의 구조를 알아봅시다.

윈도우에서는 C:\가, 리눅스에서는 /가 최상위 폴더(디렉터리)입니다.

여러분이 윈도우에서 폴더 안에 폴더(디렉터리)를 넣고 파일을 넣듯이, 리눅스에서도 똑같이 하면 됩니다.

CD는 Change Directory의 약자로, 직업 디렉터리의 위치를 바꾸는 명령입니다.

즉, 한 폴더에서 다른 폴더로 이동할 때 사용하는 명령어입니다.

CD의 기본적인 문법은 : cd [directory] 입니다.

 

  1. cd /

현재 디렉터리를 / 로 변경합니다. 여기서 / 는 최상위 경로, root를 뜻합니다. 윈도우의 C:\ 와 비슷하다고 생각하면 됩니다.

  1. cd /usr/bin

현재 디렉터리를 /usr /bin 디렉터리로 변경합니다.

  1. cd .

현재 디렉토리를 뜻하는 . 으로 이동하라는 명령어입니다. 별다른 의미는 없겠죠?

  1. cd ..

상위 디렉토리를 뜻하는 .. 으로 이동하라는 명령어입니다.

  1. 절대경로와 상대경로

현재 우리가 /usr /local /share /man / 에 있다고 해봅시다. man 디렉터리 안에 있는 man2 디렉터리로 어떻게 들어갈까요?

 
1. cd /usr/local/share/man/man2
2. cd ./man2 
3. cd man2
4. cd ../man/man2

1번 방법은 절대경로라고 부릅니다. 가장 위인 root(/)에서 시작해 목표한 디렉터리까지 내려가 줍니다. 2,3 번 방법은 상대경로라고 부릅니다. 현재 경로에서 시작하는 경로를 알려주면 됩니다.     . 은 현재 디렉터리를 뜻하는 커맨드입니다. 3번처럼 . 을 생략해도 잘 알아듣습니다. 4번도 상대경로입니다. 상위 디렉터리를 뜻하는 .. 로 상위 디렉터리인 /usr /local /share로 가준 뒤에, 다시 man으로 들어가고 man2로 들어가 결과적으로는 man2 디렉터리에 들어갑니다.

리눅스에서 파일이나 폴더(디렉터리)를 표시할 때는 절대경로, 상대경로를 둘 다 쓸 수 있는 상황이 대부분이니 상황에 따라 알맞게 쓰시면 됩니다. {.is-info}

 

  1. cd -

현재 디렉터리로 오기 전의 디렉터리로 이동합니다. 즉, 뒤로가기 입니다.

 

  1. cd ~

리눅스에서 물결표는 홈 디렉터리를 뜻합니다. 홈 디렉터리는 사용자가 시스템에 로그인 한 후 처음 오는 디렉터리입니다. ~ 는 현재 사용자의 홈 디렉터리를 나타내는 짧고 물결 모양의 문자입니다. 이렇게 치면 자신의 홈 디렉터리로 갑니다. 대부분 리눅스에서는 기본값으로 홈 디렉터리는 /home /자신의ID 디렉터리입니다.

 

ls

ls는 list의 약자로, 현재 디렉토리에 있는 파일이나 폴더를 보여주는 명령어입니다.

ls의 기본적인 문법은 : ls [option] [files] 입니다.

사용 예시는 아래와 같습니다.

  1. ls

옵션을 지정하지 않고 단순히 ls만 입력한다면, 현재 디렉터리 안에 있는 파일 및 디렉터리의 이름만 나열합니다.

 

  1. ls -l

여기에서 ls -l (-l은 숫자가 아닌 소문자 L임)은 파일 또는 디렉토리, 크기, 수정 날짜 및 시간, 파일 또는 폴더 이름 및 파일 소유자와 권한을 표시합니다.

 

  1. ls -a

-a 옵션은 . 로 시작하는 숨김 파일을 포함한 모든 파일을 나열하라는 옵션입니다.

리눅스에서는 파일 이름이 . 으로 시작하면 전부 숨김 파일입니다. ( .vimrc, .nanorc )

 

  1. ls -al

파일의 각종 정보를 나열하는 옵션 '-l'과, 숨김파일을 출력해주는 옵션 '-a'를 결합하여 '-al'과 같이 사용할 수 있습니다. 옵션 -al은 숨김파일의 자세한 정보까지 모두 출력해줍니다.

 

  1. ls [option] [files]

파일 단 한개의 정보를 보고 싶을때, ls -al filename 과 같이 입력하면, filename의 정보만 출력합니다.

  1. ls [option] [directory]

한 디렉토리의 정보를 보고 싶을때, ls -al /usr/local/ 과 같이 입력하면 /usr/local/ 디렉터리 안의 정보를 출력합니다.

 

rm

rm은 remove의 약자로, 말 그대로 파일이나 폴더를 지우는 명령어입니다.

rm의 기본적인 문법은 : rm [option] [files] 입니다

사용 예시는 아래와 같습니다.

  1. rm [files]

files을 지웁니다. 디렉토리는 지워지지 않습니다.

 

  1. rm -r [directory]

디렉토리와 그 안에 있는 디렉토리 및 파일을 모두 지웁니다. 폴더 하나를 삭제하는 것이죠.

 

  1. rm -rf [files/directory]

파일이나 디렉토리를 무조건 삭제합니다. 가끔씩 뜨는 경고문을 뜨지 않게 해주는 기능이 -f입니다.

 

 

pwd

pwd는 print working directory의 약자로, 현재 작업 디렉토리 경로를 출력합니다.

 

 

 

ln

ln은 link의 약자로, 심볼릭 링크를 생성하는 명령어 입니다.

기본적인 문법은: ln [option] [target] [file]

A라는 파일(또는 폴더)을 가리키고 있는 심볼릭 링크파일 B를 만들고 싶으면,

ln -s A B 이라고 입력하면 됩니다. 만들어진 B를 실행하면 A가 실행되고, 거의 모든 경우에 B가 A를 대체할 수 있게 됩니다.

거의 모든 상황에 그렇듯이 A,B에 들어가는 경로는 절대경로, 상대경로 둘 다 가능합니다.

 

mkdir

mkdir은 making directory의 약자로, 디렉토리를 생성하는 명령어입니다.

기본적은 문법은: mkdir [option] [path of directory] 입니다.

 

현재 내가 /usr/home/kinux98 이라는 경로에 위치해 있고, 하위 디렉토리로 candymate라는 폴더를 만들고 싶을 때, mkdir candymate 를 입력하면 하위 디렉토리로 candymate라는 폴더가 생깁니다.

또는, 절대경로를 입력하여

mkdir /usr/home/kinux98/candymate

와 같이 한 번에 입력할 수도 있습니다.

 

 

cat

cat은 concatenate 의 약자로, 파일의 내용을 출력해줍니다.

기본적인 문법은: cat [path of file] ...입니다.

현재 디렉토리에 physicube라는 텍스트(txt) 파일이 있다고 합시다. 그 파일을 읽고 싶을 때,

cat physicube 를 입력하면 파일을 읽을 수 있습니다.

이 역시 절대경로로 입력해도 똑같이 파일을 읽을 수 있습니다.

 

 

mv

mv는 move의 약자로, 한 파일을 다른 경로로 이동시키거나 이름을 바꾸는 명령어입니다.

기본적인 문법은 : mv [file] [file] : /이름 변경/ 이거나 mv [file] [dir] : /위치 변경/ 입니다.

현재 디렉토리에 posjsh라는 텍스트(txt) 파일이 있다고 합니다. 이 파일의 이름을 suhyeon이라고 바꾸고 싶을 때,

mv posjsh suhyeon 을 입력하면 됩니다.

또는, posjsh라는 파일을 상위 폴더로 옮기고 싶을 때,

mv posjsh ../ 을 입력하면 됩니다.

 

 

리눅스의 디렉토리

 

1. 리눅스 디렉토리 구조

리눅스에서는 사람들이 일반적으로 디렉토리에 대해 이야기합니다. 다른 운영 체제에서는 사람들이 유사한 개념인, 폴더를 사용합니다.

 

Linux 디렉토리 구조는 마치 나무와 같습니다.

 

 

Linux 파일 시스템은 / (루트) 에서 시작됩니다. Linux의 디렉토리 구분 기호는 슬래시 (/) 입니다.

 

다음은 가장 일반적인 최상위 디렉토리들이 대충 어떤 것인지에 대한 내용입니다.

  • / - "루트, root" , 파일 시스템 계층 구조의 맨 위. 모든 것이 루트부터 시작됩니다.

    • /bin - 바이너리 및 기타 실행 프로그램 디렉토리

    몇몇 실행 가능 프로그램은 /bin에있는 경우가 있습니다.

    • /etc - 시스템 구성 파일 디렉토리.

    운영 체제 또는 응용 프로그램의 설정 파일입니다. 예를 들어 /etc에 텍스트 모드 또는 그래픽 모드로 부팅할지 여부를 운영 체제에 알려주는 구성 파일이 있습니다.

    • /home - 홈 디렉토리.

    리눅스 시스템은 종종 여러 사용자를 가질 수 있으며 홈 디렉토리는 내 계정에 귀속된 공간입니다. 예를 들어, 사용자 kinux98의 홈 디렉토리는 /home /kinux98 입니다.

 

2. 상대경로 VS. 절대경로

 

 

 

1. 상대경로

상대 경로는 현재 작업 디렉토리 (pwd)와 관련된 경로로 정의됩니다.

 

내가 현재 /home /tux 에 있다고 가정하고 /home /yxz 로 디렉토리를 변경하려고합니다.

이때, cd를 이용하여 이동할 수 있습니다.

 

  1. 우선, /tux의 상위폴더인 /home으로 이동하기 위해 cd .. 를 입력합니다.
  2. 그럼, 현재 작업 디렉토리는 /home이 됩니다.
  3. /home의 하위폴더인 yxz로 이동하기 위해 cd yxz/ 를 입력합니다.
  4. 그럼, 현재 작업 디렉토리는 /home/yxz가 됩니다.

 

성공적으로 이동을 완료했습니다 !

이 동작을 cd ../yxz/ 로 한 번에 할 수도 있습니다.

 

2. 절대경로

절대 경로는 루트 디렉토리 (/)에서 파일 또는 디렉토리의 위치를 지정하는 것으로 정의됩니다.

즉, 절대 경로는 '/' 디렉토리에서 실제 파일 시스템의 위치까지를 나타내는 완전한 경로라고 말할 수 있습니다.

 

내가 /home /tux 에 있다고 가정하고 /home/yxz 로 디렉토리를 변경하려고합니다.

이때, cd를 이용하여 이동할 수 있습니다.

 

  1. 내가 이동하고자 하는 경로가 /home/yxz이므로 절대경로를 이용하여 한 번에 이동하기 위해

    cd /home/yxz/ 를 입력합니다.

  2. 그럼, 현재 작업 디렉토리는 /home/yxz가 됩니다.

 

성공적으로 이동을 완료했습니다 !

 

이제 이 둘의 차이를 아시겠나요?

앞에서 설명한 많은 명령어들을 사용할 때, 상대경로 또는 절대경로로 파일 경로를 지정할 수 있습니다 !

 

 

 

쉘(shell)이란?

쉘은 쉽게 표현하면 로그인 이후부터 사용자가 입력하는 명령어를 커널에 전달하는 역할을 하는 명령어 해석기를 말합니다.

 

무슨 말인지 모르겠다고요?

Shell이란 한마디로 사용자와 운영체제 사이를 연결해주는 하나의 유틸리티 프로그램 입니다.

내가 리눅스와 소통을 할 수 있게 리눅스 커널 (리눅스OS)과 나를 연결시켜주는 프로그램 이지요.

 

 

 

아까 명령어를 배울 때, 내가 cd candymate를 입력하면 candymate라는 폴더로 들어갈 수 있었습니다.

cd candymate라는 명령어는 kernel(OS)을 거쳐 cpu로 가게 됩니다. 그 결과가 다시 kernel(OS)을 거쳐 shell로 나오는 것이죠.

이러한 shell은 다양한 종류가 개발되어있습니다. 그 중 대표적인 두 가지만 소개하겠습니다.

 

 

1. BASH

 

배시 (Bash, Bourne-again shell, 본 어게인 셸)는 자유 소프트웨어로서 GNU 프로젝트를 위해 브라이언 폭스(Brian Fox)가 작성한 유닉스 셸입니다. 리눅스에 기본적으로 설치되어있는 쉘이며, 가장 광범위하게 사용되는 쉘이기도 합니다.

보통 여러분들이 처음 리눅스를 접하면 디폴트로 실행되는 쉘입니다.

 

2. ZSH

Z 셸(Z shell, zsh)은 상호작용 로그인 셸이자 셸 스크립트를 위한 강력한 명령 줄 인터프리터로 사용할 수 있는 유닉스 셸입니다. Zsh는 bash, ksh, tcsh의 일부 기능을 포함하여 수많은 개선 사항이 갖추어진 확장형 본 셸이기도 합니다.
